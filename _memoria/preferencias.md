# Preferências de Desenvolvimento — AuditCom

## Padrões de Código e Stack
- **Backend:** Node.js (API, orquestração) e Python (IA, processamento local, automação de auditoria).
- **Frontend:** HTML e Javascript Vanilla para lógica do cliente.
- **Styling (CSS):** Vanilla CSS para máxima flexibilidade. Proibido TailwindCSS (a menos que expressamente solicitado).
- **Banco de Dados:** Supabase para persistência de dados de auditorias, leads e status.
- **Infraestrutura:** Docker Compose local para desenvolvimento integrado.

## Regras Visuais e Design System (AuditCom)
- **Fundo Principal (Escuro):** `#0B1D2A` (Azul-marinho profundo).
- **Fundo Secundário / Gradientes:** `#0E3D47` (Azul petróleo).
- **Cor de Destaque / CTA:** `#00A878` (Verde-turquesa — assinatura da marca).
- **Cor de Apoio / Detalhes Premium:** `#C89D5B` (Dourado/mostarda).
- **Textos Escuros / Neutros:** `#2B3137` (Cinza-chumbo).
- **Textos Secundários:** `#8A9096` (Cinza médio).
- **Fundo Claro / Contraste / Texto Claro:** `#FFFFFF` (Branco).
- **Tipografia:**
  - Títulos e Corpo: `'Montserrat'` (pesos Regular, Medium, Semibold).
- **Restrição de Emojis:** Proibido o uso de emojis em qualquer interface de usuário comercial, PDF de relatório, carrossel de entrega ou documentos oficiais.

## Filosofia Operacional
- **Método Zero Trabalho Burro:** Identificar e automatizar toda tarefa manual ou burocrática repetitiva. O foco é a eficiência invisível da máquina trabalhando para os sócios.
